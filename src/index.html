Project developed for the Graph Theory course at Universidade Federal de Sao Carlos. The application of fundamental algorithms in Graph Theory.

Authors: L. O. David, L. Yamanaka.

Algorithms

Prim's Algorithm

Let G=(V, E) be a graph and E a set of edges with arbitrary weights contained in the Real interval. Prim's algorithm will extract the Minimum Spanning Tree (Sedgewick appud Feofiloff, IME-USP, 2012), i.e., the acyclic subgraph such that the sum of all edges weights is minimum.
The construction of the MST is done by selecting the edge with minimum weight between the set of edges adjacent to the previously expanded nodes, as long as this selection does not imply on the creation of a cycle.

Kruskal's Algorithm

Similarly to Prim's algorithm, Kruskal's goal is also to extract a MST from a graph. However, it will do so by removing the edge with smallest weight between all edges not yet evaluated (differently from Prim's, which only considers the ones that touch the expanded nodes).

Difficulties Found

We considered Kruskal's as the most difficult algorithm to implement. Working with sets of nodes to prevent cycles implied in the writing of many lines of code. We concluded that our Kruskal's implementation presents significant less performance than Prim's.
We understand, however, Kruskal's efficacy when dealing with disconnected graphs: it will successfully extract a forest of MST's of all non-connected communities, whereas Prim's would only be able to extract a single tree, possibily presenting different answers when starting from different nodes.

Breadth-First Search

BFS will transverse the graph and extract a path with fewest hops between the starting node and all the others. BFS trees are characteristically short, and contain many cluster nodes.
BFS disconnected graphs will - without doubt - leave one or more nodes out of the resulting tree.
BFS was considered by the group as the easiest - and shortest - algorithm to implement.

Depth-First Search

Like BFS, DFS will also produce a tree. This tree, however, will have a very peculiar shape: as the node's children at the most-left - or most-right - are aways recursively expanded first, the algorithm will produce tall trees, most likely with very few clusters.
This approach has many applications, such as maze solving.

Dijkstra's Algorithm

Dijkstra's many goal is to extract a Shortest-path Tree. That is, a acyclic subgraph which represents the shortest path (in this context, shortest is equivalent to smaller weight sum) between the origin to all the other nodes.

Welch and Powell's Algorithm

Given a sequence of colors, Welch e Powell's algorithm will attempt to paint each node of the graph in such a way that no adjacent nodes are painted in the same colors. Furthermore, the smallest number of colors will be used.
Welch and Powell's is broadly used for solving collision problems.
Obvisouly, this doesn't apply to graphs with nodes that are connected to themselves.

Graphical Features Developed

Node Radius Proportional to Connectivity

Initially, the group thought it would be interesting to represent the connectivity of the nodes by their sizes. Nodes strongly connected (hubs) would present bigger sizes and nodes weakly connected, smaller ones.

Difficulties Found

We observed a great difficulty in maintaing reasonable proportions in the nodes's sizes, after implementing this feature. In order to overcome this problem, we decided that the size should be a constant scalled by the rate between the node connectivity and the average graph connectivity.

Coloring

In order to demonstrate Welch and Powell's algorithm, a feature was implemented to re-draw the entire graph, but considering an additional color property.
